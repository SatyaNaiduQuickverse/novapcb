# novapcb-layout — Phase 4 board (Phase 4a scaffolding onward)

KiCad 9 project for the novapcb v1 PCB layout. Generated by
`generate_board.py` from `../novapcb/novapcb.net` (the Phase 3 netlist).

## Regenerate

```bash
KICAD9_FOOTPRINT_DIR=/usr/share/kicad/footprints python3 generate_board.py
```

Reads the netlist; produces `novapcb-layout.kicad_pcb` deterministically
(bit-identical modulo KiCad's internal UUIDs).

`novapcb-layout.kicad_pro` is hand-maintained (net classes + DRC severity
levels — these live in the project JSON, not the .kicad_pcb, per KiCad 9
convention).

## What this project contains (Phase 4a deliverable)

- 70 footprints assigned (all from Phase 3 netlist + 8 ESC pads from the
  in-repo `novapcb_lib`); no placeholders
- 4-copper-layer stackup (DECISIONS §8)
- Closed 36 × 36 mm Edge.Cuts outline (single SHAPE_T_RECT primitive)
- 4× M3 mounting holes at 30.5 × 30.5 mm c-to-c (corners; 2.75 mm inset)
- DRC ruleset: 0.13 mm min track / clearance; 0.45 mm min via; 0.30 mm
  edge clearance (JLCPCB 4-layer-capable)
- Net classes: Default, Power_3V3, Power_5V, Power_VBAT, USB_diffpair,
  IMU_SPI, SDMMC, DShot
- Netclass patterns map +3V3 → Power_3V3, MOT* → DShot, SPI1_* → IMU_SPI,
  SDMMC1_* → SDMMC, USB_D[PM] → USB_diffpair, etc.

## What it does NOT contain

- Placement (Phase 4b)
- Copper planes (Phase 4c)
- Routing (Phase 4d/4e)
- Final gerbers/drill (Phase 4f)

## In-repo footprint library

`lib/novapcb.pretty/` — committed, project-bound (`${KIPRJMOD}/lib/...`):

- `ESC_solder_pad.kicad_mod` — 2× SMD pads, 2.0×1.5 mm at 2.5 mm pitch
  (signal + GND). MatekH743-style mini-FC ESC convention. Replaces the
  Phase 3f `PinHeader_1x02_P2.54mm_Vertical` placeholder.

See `../PHASE4_P0_REPORT.md §P0.4` for the full design-rule rationale and
`docs/PHASE3_AUDIT.md §B` for the carry-forward items this 4a addresses.
